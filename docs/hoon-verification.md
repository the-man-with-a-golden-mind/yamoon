# Verifying Yamoon Code in Urbit

To ensure that the Hoon code generated by yamoon is functional and correct, you can use the built-in verification pipeline.

## 1. Prerequisites

*   A local Urbit pier (development ship).
*   Yamoon CLI installed.

## 2. The Verification Pipeline

The `yamoon sync` command compiles your project and automatically copies the generated Hoon files into your Urbit pier's file system.

### Step 1: Sync with your Pier

Run the following command, replacing `<pier_path>` with the absolute path to your Urbit pier:

```bash
yamoon sync examples/basic.hyml /path/to/my-pier
```

This will:
1.  Compile `basic.hyml` into `lib/basic.hoon`.
2.  Generate a Hoon test file at `tests/lib/basic.hoon`.
3.  Copy both to your pier's `base` desk.

### Step 2: Run Urbit Tests

Inside your Urbit dojo, run the following commands:

```hoon
:: Mount the base desk if not already mounted
|mount %base

:: Run the test suite
-test /=base=/tests/lib/basic
```

## 3. Automated Script

You can use the provided helper script to automate the sync and display the test command:

```bash
./scripts/verify_hoon.sh examples/basic.hyml /path/to/my-pier
```

## 4. Why this matters

Hoon is a unique, low-level language. While yamoon's compiler is type-checked and rigorous, running the generated code in a real Urbit environment ensures that:
*   **Standard Library Compatibility**: Mappings to `turn`, `snoc`, `by`, and `in` engines are correctly utilized.
*   **Gall Door Semantics**: The generated agent correctly interfaces with the Arvo kernel.
*   **Performance**: The generated Hoon is idiomatic and follows irregular form optimizations.
